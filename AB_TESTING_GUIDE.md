# A/B Testing Preview Guide

## Bramhollow Condominium Association Newsletter System

This guide explains how to use the A/B Testing feature in the Newsletter Generator to compare different versions of your newsletter content before sending.

---

## What is A/B Testing?

A/B Testing allows you to create two different versions of your newsletter and compare them side-by-side. This helps you:

- Choose the most effective subject line or headline
- Compare different content layouts
- Test various writing styles
- Get feedback from colleagues before sending
- Make data-driven decisions about your communications

---

## Getting Started with A/B Testing

### Step 1: Enable A/B Testing

1. Open the **Newsletter Generator** from the Dashboard
2. Look for the **"A/B Test"** button in the top toolbar
3. Click the button to enable A/B Testing mode
4. A new window will open with the A/B Testing interface

### Step 2: Create Your Versions

When you enable A/B Testing, your current newsletter content is automatically copied to both Version A and Version B. From there, you can:

1. Select which version to edit using the sidebar buttons
2. Modify the content in the markdown editor
3. Choose different templates for each version
4. Give each version a descriptive name

---

## Using the A/B Testing Interface

### Edit Versions Tab

This is where you create and modify your newsletter versions:

**Version Selector (Left Sidebar)**
- Click **Version A** or **Version B** to switch between them
- Each version shows a colored badge (blue for A, green for B)

**Version Settings**
- **Version Name**: Give each version a descriptive name (e.g., "Formal Tone" vs "Casual Tone")
- **Template**: Select different email templates for each version

**Editor Area**
- The left side shows the markdown editor
- The right side shows a live preview of the email

### Compare & Vote Tab

This is where you compare both versions side-by-side and collect votes:

**Layout Options**
- **Side-by-Side**: Shows both versions next to each other
- **Stacked**: Shows versions one above the other

**Voting System**
- Click **"Vote A"** or **"Vote B"** to record a preference
- Share your screen with colleagues to collect their votes
- The progress bar shows the current vote distribution
- Use **"Reset Votes"** to start fresh

**Version Actions**
- **Copy**: Copy the HTML for that version to your clipboard
- **Export**: Download the HTML file for that version
- **Vote**: Record a vote for that version

---

## Selecting a Winner

Once you've compared both versions and collected feedback:

1. Go to the **Compare & Vote** tab
2. Review the voting results at the top
3. Look at the bottom of the screen for the winner selection buttons
4. Click **"Use Version A"** or **"Use Version B"**
5. The winning version becomes your main newsletter content
6. A/B Testing mode is automatically disabled

## Best Practices for A/B Testing

### What to Test

**Headlines and Subject Lines**
- Try different word choices
- Test emotional vs. factual approaches
- Experiment with length

**Content Structure**
- Lead with important news vs. general updates
- Bulleted lists vs. paragraphs
- Short sections vs. detailed content

**Tone and Style**
- Formal vs. conversational
- First person vs. third person
- Active vs. passive voice

**Call to Action**
- Different button text
- Urgency vs. casual requests
- Placement within the content

### Tips for Better Testing

1. **Change One Thing at a Time**: Make only one significant change between versions to identify what works best

2. **Get Multiple Opinions**: Share your screen and ask board members or colleagues to vote

3. **Consider Your Audience**: Think about what resonates with Bramhollow residents

4. **Test Before Major Announcements**: Use A/B testing for important communications like dues notices or meeting announcements

5. **Document What Works**: Keep notes about which approaches get better responses

## Exiting A/B Testing

You can exit A/B Testing mode at any time:

1. Click the **"Exit A/B Test"** button in the top right corner
2. Confirm your action
3. You'll return to the standard newsletter editor

**Note**: If you exit without selecting a winner, your current main editor content remains unchanged.

---

## Keyboard Shortcuts

While in the A/B Testing interface:

- **Tab**: Switch between Edit and Compare tabs
- **Ctrl/Cmd + C**: Copy selected text
- **Ctrl/Cmd + V**: Paste text

## Troubleshooting

### Content Not Showing in Preview

- Make sure you have content in the markdown editor
- Check that your markdown syntax is correct
- Try refreshing the page if the preview is stuck

### Votes Not Recording

- Votes are stored in your browser session
- If you refresh the page, votes will be reset
- Use the voting feature during a single session

### Can't Exit A/B Testing

- Click the red "Exit A/B Test" button
- If the button is not visible, scroll to the top of the modal
- Close the modal by clicking outside it or pressing Escape
